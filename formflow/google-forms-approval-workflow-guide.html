Build a Google Forms approval workflow with email approval and Sheet status

A Google Forms approval workflow routes each submission to a reviewer, sends an approval email, records approve/reject decisions, and updates the linked Google Sheets response file.

Install FormFlow Run first workflow test

Quick answer

Use Google Forms for the request, the linked Google Sheet for the workflow record, and FormFlow for reviewer emails, approve/reject links, Preflight, and approval status tracking.

Best fit

Lightweight operational approvals

Use it for leave requests, access requests, event applications, resource requests, and internal submissions that need one clear decision path.

Not for

Enterprise workflow platforms

Do not use it as a procurement suite, ticketing system, CRM, legal approval platform, or compliance system.

Start with a scenario, not the broad keyword

A broad approval workflow page is hard to win and often attracts the wrong expectations. FormFlow should be evaluated through concrete Google Forms jobs: one requester, one response sheet, one reviewer email, one approve/reject decision, and one visible status column.

Leave request

Simple manager decision

Collect date range, reason, requester email, and manager context, then write Approved or Rejected beside the response.

Access request

Account or permission review

Collect who needs access, which system, and why, then route the request to the right reviewer.

Purchase request

Small operations approval

Collect amount, item, vendor, and justification, then route one email decision before spending happens.

Application review

Program or event selection

Use the linked Sheet as the review record for submissions that need one final decision.

Manual workaround

Forward each response manually

You can watch the response sheet, copy the submitted details, email reviewers, and write the result back into a status column. This works for very low volume, but it is easy to miss a request or lose the decision trail.

Build Apps Script yourself

A custom script can send approval emails and update the sheet, but someone must maintain authorization, triggers, email templates, error handling, and reviewer links.

FormFlow setup path

  1. Open the Google Form that collects the request details.
  2. Install FormFlow from Google Workspace Marketplace.
  3. Open FormFlow from the form editor and select the linked response spreadsheet.
  4. Add one internal approver email address for the first workflow.
  5. Save the approval configuration.
  6. Run Preflight and fix any missing sheet, email, or configuration warning.
  7. Submit one internal test response and confirm the reviewer email arrives.
  8. Click approve or reject, then verify the linked Sheet status changed.

What to include in the form

Requester

Email or name

Collect enough context for a reviewer to know who made the request and where the result should go.

Request

Clear decision fields

Ask for the item, amount, date, reason, or document that the reviewer needs to approve or reject.

Reviewer

Simple approver rule

Start with one approver first. Add more routing only after the first approval path works reliably.

Status

Linked Sheet record

Keep the response sheet as the operational record so support can diagnose status and email issues.

Limits and cautions

  • Test with copied or internal data before routing production requests.
  • Approval emails depend on the Google account and Workspace email environment.
  • Google Forms is not a live transaction database; keep workflows small and auditable.
  • Use support before requesting a review if email delivery, Preflight, or Sheet status does not match the expected result.

If the workflow does not complete

Preflight

Configuration issue

Confirm the linked response Sheet, saved approver email, and form-to-sheet connection before submitting another response.

Email

Delivery issue

Check the approver address, spam folder, Workspace mail rules, and whether Preflight reported any send-mail warning.

Decision

Reviewer account issue

Ask the reviewer to use the intended Google account. Multi-account browsers can make decision links look broken.

Status

Sheet update issue

Confirm the selected response spreadsheet is linked to the exact form submission being tested.

FAQ

Can I create an approval workflow with Google Forms?

Yes, but Google Forms alone only collects submissions. FormFlow adds reviewer emails, approve/reject decisions, and linked-sheet approval status.

Should I test with live requests?

No. Start with one internal test response so you can confirm email delivery, decision links, and status tracking before launch.

When should I ask for a Marketplace review?

Ask only after Preflight passes, the reviewer email arrives, the approve or reject link works, and the linked Sheet status updates. If anything is unclear, send support context first.

Is FormFlow a replacement for procurement software?

No. It is a lightweight Google Forms approval path, not a procurement, expense, ticketing, CRM, legal, or compliance suite.

What should my first approval form include?

Collect requester identity, request details, decision context, and a linked response Sheet. Keep the first version simple enough to test with one reviewer.
