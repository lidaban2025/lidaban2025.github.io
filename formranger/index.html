Failure causes

Why Google Forms choices from Sheets usually fail.

Most failed first runs are not mysterious. They come from a stale mapping, a changed Sheet column, a copied form, or checking the editor instead of the respondent-facing preview.

Source list

The Sheet column moved or changed.

A renamed header, inserted blank row, duplicate-only values, hidden cleanup column, or changed sheet tab can make the mapping point at the wrong data.

Question

The form question no longer matches.

If the dropdown was duplicated, deleted, renamed, or changed from dropdown to another question type, re-select the exact question before updating.

Action

A saved mapping is not an update.

The useful result is Update now reporting that a question changed. Saving configuration only proves the setup was stored.

Preview

The editor is not the final proof.

Always open the public respondent preview after Update now. That is the surface respondents will see.

Boundary

Not a booking or inventory lock.

FormRanger updates Google Forms choices from selected spreadsheet data before respondents use the form. It does not reserve seats, process payments, lock inventory, or filter options while a respondent is already filling out the form. If you need live respondent-time logic, a Google Form is usually the wrong surface.

Frequently asked questions

Which question types work?

Dropdown, multiple choice, checkbox, multiple-choice grid, and checkbox grid choices.

What should I test first?

Use one copied form, one supported question, and one Sheets column with Alpha and Beta. Run Preflight, update, then check the public respondent preview.

Can it handle dependent dropdowns?

Use FormRanger for spreadsheet-backed choice refresh before respondents use the form. It is not respondent-time conditional filtering.

Why did the choices not change after setup?

Most often the mapping was saved but Update now was not run, the source column moved, the question was changed or duplicated, or the public preview was not refreshed after the update.

Should I use this on a live form first?

No. Use a copied form first so you can verify Preflight, Update now, and public preview without risking real respondents.
