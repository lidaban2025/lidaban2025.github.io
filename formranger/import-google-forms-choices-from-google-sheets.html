Import Google Forms choices from Google Sheets

Importing Google Forms choices from Google Sheets means using a spreadsheet column as the source for a form question's dropdown, multiple-choice, checkbox, or grid choices. FormRanger is a Google Forms add-on that helps form owners select a Sheets source, map one question, run Preflight, and update the form choices before checking the public preview.

Quick answer

Google Forms does not natively keep a question's choices linked to a Google Sheets range. If the list rarely changes, paste the choices manually. If the list lives in Sheets and needs repeatable updates, install FormRanger, select the source sheet, map one supported question, run Preflight, click Update now, and confirm the public preview shows the imported choices.

Manual paste

Best for a one-time list where one owner can safely copy values into the form editor before launch.

FormRanger

Best when the source list stays in Sheets and the form choices should be refreshed from that selected source.

Apps Script

Useful for custom logic, but it requires code ownership, trigger maintenance, and careful permission review.

Separate form builder

Consider a different tool when respondents need live dependent filtering, booking rules, payments, or inventory locking.

Safe first-run test

  1. Create a copied test form with one dropdown, multiple-choice, checkbox, or grid question.
  2. Create a small Google Sheets column with two obvious values, such as Alpha and Beta.
  3. Install FormRanger from Google Workspace Marketplace and open it from the Google Forms Add-ons menu.
  4. Select the spreadsheet, choose the sheet tab and source column, then link one form question.
  5. Save the mapping, run Preflight, and make sure it reports at least one valid mapping.
  6. Click Update now and check that the update report says the question was updated with the expected choice count.
  7. Open the public preview and confirm the form shows the same values from the sheet.
A successful first run is not just a saved mapping. Treat the setup as complete only after the public preview shows the imported choices.

When this page is the right path

Roster lists

Teams, staff names, locations, departments, rooms, or other lists already maintained in Sheets.

Event options

Sessions, tracks, workshops, campuses, pickup windows, or categories that change before launch.

Catalog choices

Product names, item labels, package types, or service choices where the sheet is easier to maintain.

Cleanup before launch

Use sorting and duplicate removal so respondents see a cleaner choice list than the raw source column.

FormRanger updates supported Google Forms choice questions from selected Sheets data. It is not a booking system, payment workflow, inventory lock, or live respondent-time dependent dropdown engine.

After the import works

Move from the copied test form to your real form only after one question updates cleanly. If the public preview is wrong, use the support path first instead of asking users to submit the live form.

If it works

Repeat the same mapping pattern on the real form and keep the source sheet clean.

If it fails

Capture the question type, selected sheet tab, source column, Preflight text, Update now report, and preview result.

If choices are stale

Check whether the source values changed, the column was renamed, or the saved mapping points to an old tab.

If users need confidence

Share the setup guide and permissions page before they install the Marketplace add-on.

FAQ

Can I import choices without code?

Yes. Manual paste works for rare updates, while FormRanger gives a no-code sidebar flow for selected Sheets data.

Does it update live while a respondent is answering?

No. Treat FormRanger as a controlled choice-sync workflow, not live respondent-time filtering.

What source format is easiest?

Use one clean column with a clear header and visible values. Avoid merged cells and hidden cleanup logic for the first test.

When should I ask for a review?

Only after the mapping, Preflight, Update now report, and public preview all look correct.
