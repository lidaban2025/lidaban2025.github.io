After install

Installed a Google Workspace add-on but cannot find it?

Start by refreshing the file, checking the account that installed the add-on, and opening it from the correct Google app. FormSuite add-ons are split between Google Forms and Google Sheets, so the right first click depends on the product you installed.

Find the add-on Install center First-run checklist Send setup note

Quick answer

After installing from Google Workspace Marketplace, reload the current file. Then open the add-on from the host app it was built for. A Forms add-on will not appear inside a Sheet, and a Sheets add-on will not appear inside a Form.

Google Forms

FormGuard, FormNotifier, FormRanger

Open the Google Form editor, refresh the tab, then use the Add-ons puzzle-piece menu or the add-on entry shown in the Forms editor.

Google Sheets

FormCopy, FormMerge, DocForge

Open the target spreadsheet, refresh the tab, then check the right sidebar and Extensions > Add-ons for the installed add-on.

Same account

Confirm the installer account

Use the same Google account that installed the Marketplace app. If a Workspace admin installed or blocked apps, the menu state may differ by account.

Step-by-step check

  1. Open the exact Google Form or Google Sheet you want to use for the first test.
  2. Refresh the browser tab after installation. If you installed from another tab, reload the file before looking for the add-on.
  3. Confirm you are signed in with the same Google account that installed the add-on.
  4. For Forms add-ons, open the Google Forms editor and check the Add-ons menu.
  5. For Sheets add-ons, check the right sidebar first, then Extensions > Add-ons.
  6. If the add-on still does not appear, open the Marketplace listing again and confirm it says installed, or open Marketplace Manage apps.
  7. If you use a school or company account, check whether your Workspace admin restricts Marketplace apps.
Google's help docs also recommend refreshing the file when an add-on does not appear after install, and describe installed apps through Marketplace Manage apps and the Extensions > Add-ons menu.

If it still does not show up

Try a new test file

Create a blank test Form or Sheet and look for the add-on there. This separates account/install issues from one file's state.

Check browser state

Disable extensions that block sidebars or popups, and try a normal Chrome profile where you are signed in to the correct account.

Send a setup note

Include the add-on name, host app, account type, where you looked, and whether Marketplace shows the app as installed. Do not include private form responses or sheet data.

After it opens

Do not start with a real batch. Run one small first proof, confirm the visible result, then leave a short Marketplace review only if the first test worked.
