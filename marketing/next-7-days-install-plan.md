# FormSuite next 7 days install plan

_Updated: 2026-05-31_

Goal: get the first real installs again. Do not wait for SEO. Use high-intent manual distribution and Marketplace iteration.

## Positioning for this week

### FormGuard

Do **not** lead with generic “Google Forms response limit”. Google Forms already has native response limits for simple cases.

Lead with:

- per-choice quotas,
- appointment/time-slot capacity,
- event/session seats,
- waitlist/closed-message workflow,
- test-before-launch safety.

One-line pitch:

> FormGuard helps Google Forms users stop overbooking individual choices: workshops, appointment slots, roles, sessions, and limited signup options.

### FormRanger

Lead with:

- dropdown / checkbox / multiple-choice options from Google Sheets,
- no Apps Script maintenance,
- test before launch.

One-line pitch:

> FormRanger keeps Google Forms choices synced from Google Sheets without maintaining Apps Script triggers.

## Day 2 — FormGuard choice quota threads

Search queries:

- `"Google Forms" "limit each option"`
- `"Google Forms" "choice limit"`
- `"Google Forms" "appointment slots" "limit"`
- `"Choice Eliminator" alternative`
- `"Google Forms" "workshop" "capacity"`

Reply template:

> For total response limits, Google Forms may be enough now. The harder case is option-level capacity: each workshop/session/time slot needs its own quota while the rest of the form stays open.
>
> The clean setup is:
> 1. make each session/slot a choice option,
> 2. assign a quota to each option,
> 3. test the public form with tiny sample limits,
> 4. prepare the full/waitlist message before launch.
>
> I’m building FormGuard for this exact Google Forms capacity workflow: https://formsuite.dev/formguard/google-forms-choice-limit.html

## Day 3 — FormRanger dynamic choices threads

- `"populate Google Forms" "from Sheets"`
- `"FormRanger alternative"`

Reply template:

>
> Basic workflow:
> 1. keep choices in one clean Sheet column,
> 2. map that range to the form question,
> 3. update choices,
> 4. preview the public form before launch.
>
> I’m building FormRanger around this workflow: https://formsuite.dev/formranger/dynamic-google-forms-dropdown-guide.html
