Why are there no reviews yet?

The more likely cause is not one single product bug. It is an early-launch conversion gap: low visible install counts, users leaving after setup, and not enough review prompts after a successful test.

What the current data says

FormGuard and FormNotifier have the strongest visible install base and both have recent smoke evidence for core first-run paths. FormMerge and DocForge have passed sidebar/first-use checks, but their user counts are still tiny. FormCopy has a public demo and guide path, but it still needs first visible users.

What to do next

Ask for reviews only after a real success signal: a saved rule, passed preflight, test email, generated document, or formula copy. Do not ask for five-star reviews. Ask for a short honest review describing what worked.

Best first review targets

Prioritize FormGuard and FormNotifier first because they already show the most visible users. Next, push one clear first-run path for FormMerge and DocForge. FormCopy needs install acquisition before review acquisition.

Product risk check

Keep testing, but do not assume "0 reviews" means broken. A broken product usually shows support complaints, failed smoke tests, or uninstall patterns. The current signal is more about trust, traffic, and review timing.
