Core features

FormMerge is designed for teams that need dependable operational email delivery, not a marketing tool with unrelated complexity.

Dynamic merge tags

Use column headers to personalize the subject and body of each outgoing message.

Test before send

Preview and send a safe sample before you run a larger batch or rely on the setup in production.

Batch sending

Process unsent rows in a deliberate run when you need operational control over timing.

Status visibility

Track what was sent, skipped, or failed directly inside the sheet your team already reviews.

Typical use cases

Applicant and respondent updates

Send confirmations, next steps, or instructions tied to each submitted row.

Internal team notifications

Deliver personalized operational emails to owners, reviewers, or request handlers.

Event and program operations

Communicate schedules, reminders, or follow-up actions from a well-structured source sheet.

Batch review workflows

Hold a queue of ready records in Sheets and send them on your own cadence.

Quick start

A simple setup path that stays inside Google Sheets.

  1. Open the sheet containing the rows you want to email from.
  2. Launch FormMerge and define the message template, recipients, and merge fields.
  3. Run a test send to validate the output.
  4. Send the batch and monitor the status columns in the same sheet.

Frequently asked questions

How is this different from old mail-merge scripts?

FormMerge Pro is built for current Google Sheets workflows with HTML templates, CC/BCC, test sends, and row-level status tracking.

Can I send HTML emails?

Yes. Full HTML template support with merge tags.

Should I install before asking about plans?

Yes. During launch mode, install from Marketplace and test one small row-based email workflow before any commercial conversation.

Do I need coding experience?

No. Configure templates from the sidebar inside Google Sheets.
